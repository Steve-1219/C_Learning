# 7.2 函数的调用

## 📖 知识梳理

### 7.2.1 函数调用的形式和方式

#### 一、函数调用的形式

**1. 调用无参函数**

```c
函数名()
```

**示例：**

```c
print_star();    // 调用无参函数
```

**2. 调用有参函数**

```c
函数名(实参表列)
```

**示例：**

```c
min(a, b);       // 调用有参函数，a和b是实参
```

> 💡 **拿分笔记**：如果实参表列包含多个实参，则各参数间用**逗号**隔开。实参与形参的个数应**相等**，类型应**匹配**。

#### 二、函数调用的方式

按函数调用在程序中出现的形式和位置来分，有以下3种函数调用方式：

| 调用方式 | 示例 | 说明 |
| :--- | :--- | :--- |
| **函数调用语句** | `min(a, b);` | 把函数调用单独作为一个语句，不要求函数返回值 |
| **函数表达式** | `c = 5 + min(a, b);` | 函数调用出现在一个表达式中，函数返回值参加表达式的运算 |
| **函数参数** | `printf("%d", min(a, b));` | 函数调用作为另一个函数的实参 |

> 💡 **注意**：调用函数并不一定要求包含分号，只有作为**函数调用语句**时才需要有分号。如果作为函数表达式或函数参数，函数调用本身是不必有分号的。


### 7.2.2 函数调用的过程 

#### 一、实参与形参的数据传递

**1. 形式参数和实际参数**

| 参数类型 | 位置 | 说明 |
| :--- | :--- | :--- |
| **形式参数（形参）** | 定义函数时，函数名后面括号中的变量名 | 用于接收传递过来的数据 |
| **实际参数（实参）** | 调用函数时，函数名后面括号中的参数 | 将数据传递给形参 |

**2. 实参和形参间的数据传递**

在调用函数过程中，系统会把实参的**值**传递给被调用函数的形参。或者说，形参从实参得到一个值。

> 💡 **拿分笔记**：实参向形参的数据传递是**"值传递"**，**单向传递**。只能由实参传给形参，而不能由形参传给实参。实参和形参在内存中占有不同的存储单元，实参无法得到形参的值。

**【例2】输入两个整数，要求输出其中值较小者。使用函数实现。**

```c
#include <stdio.h>

int main()
{
    int min(int x, int y);    // 对min函数的声明
    int a, b, c;
    printf("请输入两个整数：");
    scanf("%d,%d", &a, &b);
    c = min(a, b);            // 调用min函数，a和b是实参
    printf("较小数是 %d\n", c);
    return 0;
}

// 定义min函数
int min(int x, int y)         // x和y是形参
{
    int z;
    if (x < y)
        z = x;
    else
        z = y;
    return z;
}
```

**运行结果：**

```
请输入两个整数：2,3
较小数是 2
```

**数据传递过程：**

```
┌─────────────────────────────────────────────────────────┐
│                      数据传递示意图                      │
│                                                         │
│   main函数              →           min函数             │
│                                                         │
│   实参 a = 2    ────────────────→  形参 x = 2          │
│   实参 b = 3    ────────────────→  形参 y = 3          │
│                                                         │
│   ⚠️ 传递方向：单向（只能从实参传到形参）               │
│   ⚠️ 存储单元：实参和形参占有不同的存储单元              │
└─────────────────────────────────────────────────────────┘
```

**调用过程的详细说明：**

| 步骤 | 说明 |
| :---: | :--- |
| ① | 在定义函数中指定的形参，在未出现函数调用时，它们**不占用**内存中的存储单元 |
| ② | 发生函数调用时，形参被**临时分配**内存单元 |
| ③ | 将实参的**值**传递给对应的形参 |
| ④ | 执行被调用函数中的操作（如比较x和y的大小） |
| ⑤ | 将函数值（返回值）传回主调函数 |
| ⑥ | 调用结束，形参的存储单元被**释放** |
| ⑦ | 实参的值**不变**（保留并维持原值） |

#### 二、名师点金

| 要点 | 说明 |
| :--- | :--- |
| ① 形参个数 | 可以有多个，可以相同类型，也可以不同类型 |
| ② 实参类型 | 可以是**常量、变量或表达式**，如 `min(3, a + b)` |
| ③ 类型匹配 | 实参与形参的类型应**相同或赋值兼容**，否则按赋值规则自动转换 |
| ④ 值传递 | 形参值的改变**不会影响**实参的值 |

#### 三、函数的返回值

如果希望函数调用后返回一个执行结果，则函数必须使用 `return` 语句将结果返回给主调函数。

**return语句的格式：**

```c
return(表达式);
// 或
return 表达式;
```

**示例：**

```c
int min(int x, int y)
{
    return (x < y ? x : y);    // 直接返回表达式
}
```

**使用返回值时应注意：**

| 要点 | 说明 |
| :--- | :--- |
| ① 多个return | 一个函数中可以有多个 `return` 语句，遇到第一个 `return` 时立即终止函数执行 |
| ② 返回值类型 | 应与函数类型一致，不一致时自动进行类型转换 |
| ③ 省略类型 | 如果返回值类型为 `int`，函数类型说明符可以省略（建议保留） |

**多return语句示例：**

```c
int min(int x, int y)
{
    if (x < y)
        return x;      // x<y时执行，函数立即结束
    else
        return y;      // x>=y时执行，函数立即结束
}
```


### 7.2.3 对被调用函数的声明 

#### 一、调用函数的条件

在一个函数中调用另一个函数（即被调用函数）需要具备以下条件：

| 条件 | 说明 |
| :--- | :--- |
| ① **函数已定义** | 被调用的函数必须是已经定义的函数（库函数或用户自定义函数） |
| ② **库函数** | 需要使用 `#include` 指令包含对应的头文件 |
| ③ **自定义函数声明** | 如果函数定义在调用之后，需要在调用前进行**函数声明** |

**函数声明的一般形式：**

```c
类型名 函数名(形参列表);
```

**示例：**

```c
float add(float x, float y);    // 函数声明
```

**【例3】输入两个实数，用一个函数求出它们的和。**

```c
#include <stdio.h>

int main()
{
    float add(float x, float y);    // 对add函数作声明
    float a, b, c;
    printf("请输入a和b:");
    scanf("%f,%f", &a, &b);
    c = add(a, b);                  // 调用add函数
    printf("和是%f\n", c);
    return 0;
}

// 定义add函数（在main函数之后）
float add(float x, float y)
{
    return x + y;
}
```

> ⚠️ **注意**：如果函数定义在主调函数**之前**，则不需要再作函数声明。


### 7.2.4 函数的嵌套调用 

#### 一、嵌套调用的概念

C语言的函数定义是互相**平行、独立**的，即在定义函数时，一个函数内**不能再定义另一个函数**（不能嵌套定义），但**可以嵌套调用函数**——在调用一个函数的过程中，又调用另一个函数。

```
┌─────────────────────────────────────────────────────┐
│                   嵌套调用示意图                     │
│                                                     │
│   main函数         a函数           b函数            │
│                                                     │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐       │
│   │ 开始执行 │    │ 开始执行 │    │ 开始执行 │       │
│   │    ↓     │    │    ↓     │    │    ↓     │       │
│   │调用a函数 │───→│执行部分  │    │执行功能  │       │
│   │    ↓     │    │    ↓     │    │    ↓     │       │
│   │执行剩余  │    │调用b函数 │───→│ 结束    │       │
│   │    ↓     │    │    ↓     │    │    │     │       │
│   │ 结束     │    │执行剩余  │    │    │     │       │
│   └─────────┘    │    ↓     │    │    ↓     │       │
│                  │ 结束     │    │ 返回a    │       │
│                  └─────────┘    └─────────┘       │
│                                                     │
│   执行顺序：main → a → b → 返回a → 返回main        │
└─────────────────────────────────────────────────────┘
```

**嵌套调用的执行过程：**

| 步骤 | 执行内容 |
| :---: | :--- |
| ① | 执行 `main` 函数的开头部分 |
| ② | 调用 `a` 函数，流程转去 `a` 函数 |
| ③ | 执行 `a` 函数的开头部分 |
| ④ | 调用 `b` 函数，流程转去 `b` 函数 |
| ⑤ | 执行 `b` 函数（如无其他嵌套，完成全部操作） |
| ⑥ | 返回到 `a` 函数中调用 `b` 函数的位置 |
| ⑦ | 继续执行 `a` 函数中尚未执行的部分 |
| ⑧ | 返回 `main` 函数中调用 `a` 函数的位置 |
| ⑨ | 继续执行 `main` 函数的剩余部分 |

**【例4】输入4个整数，找出其中最大的数。用函数的嵌套调用来处理。**


### 7.2.5 函数的递归调用 

#### 一、递归调用的概念

在调用一个函数的过程中，**直接或间接调用该函数本身**，称为函数的递归调用。

**① 直接调用：** 在调用 `f` 函数的过程中，又要调用 `f` 函数

```
┌─────────────────┐
│   f函数          │
│   ┌───────────┐ │
│   │ 调用f函数  │ │  ← 直接调用本身
│   └───────────┘ │
└─────────────────┘
```

**② 间接调用：** 在调用 `f1` 函数过程中要调用 `f2` 函数，而在调用 `f2` 函数过程中又要调用 `f1` 函数

```
┌─────────┐       ┌─────────┐
│  f1函数  │──────→│  f2函数  │
│          │←──────│          │
└─────────┘       └─────────┘
  ↑                    ↑
  └────── 间接调用 ────┘
```

> ⚠️ **重要**：这两种递归调用都是**无终止的自身调用**，在程序中没有意义。程序中应该**限制递归调用的次数**，或者指定只有在**某一条件成立时才继续执行递归调用**，否则就不再继续。

#### 二、递归调用的应用

**【例6】用递归方法求 n! (n > 0)。**

**递归公式：**

```
       ┌ 1              (n = 1)
f(n) = ┤
       └ n * f(n - 1)   (n > 1)
```

**递归求阶乘的过程（以 n=5 为例）：**

```
5! = 5 × 4!
        │
        4! = 4 × 3!
              │
              3! = 3 × 2!
                    │
                    2! = 2 × 1!
                          │
                          1! = 1   ← 递归结束条件（n=1）
```

**程序代码：**

```c
#include <stdio.h>

int fact(int n)    // 递归函数
{
    if (n == 1)
        return 1;          // 递归结束条件
    else
        return n * fact(n - 1);    // 递归调用
}

int main()
{
    int n;
    printf("请输入n的值：");
    scanf("%d", &n);
    printf("%d! = %d\n", n, fact(n));
    return 0;
}
```

**运行结果：**

```
请输入n的值：5
5! = 120
```

**递归执行过程：**

```
fact(5) = 5 * fact(4)
              fact(4) = 4 * fact(3)
                           fact(3) = 3 * fact(2)
                                        fact(2) = 2 * fact(1)
                                                     fact(1) = 1
                                        fact(2) = 2 * 1 = 2
                           fact(3) = 3 * 2 = 6
              fact(4) = 4 * 6 = 24
fact(5) = 5 * 24 = 120
```

**递推与递归的对比：**

| 对比项 | 递推 | 递归 |
| :--- | :--- | :--- |
| **方向** | 从已知出发，向目标推进 | 从目标出发，向已知回溯 |
| **起点** | 已知的事实（如 1! = 1） | 目标问题（如 5!） |
| **过程** | 1! → 2! → 3! → ... → n! | n! → (n-1)! → ... → 1! |
| **实现方式** | 循环 | 函数自身调用 |


## 💡 核心考点

1. **函数调用形式**：无参函数 `函数名()`；有参函数 `函数名(实参表列)`
2. **调用方式**：函数调用语句、函数表达式、函数参数
3. **实参与形参**：实参传递给形参，**单向值传递**
4. **形参特点**：调用时分配存储单元，调用结束释放
5. **return语句**：返回值并终止函数执行
6. **函数声明**：`类型名 函数名(形参列表);` 定义在调用前则无需声明
7. **嵌套调用**：函数不能嵌套定义，但可以嵌套调用
8. **递归调用**：函数直接或间接调用自身，需有结束条件
