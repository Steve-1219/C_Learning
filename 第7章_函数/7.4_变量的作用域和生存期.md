# 7.4 变量的作用域和生存期

## 🎯 学习目标

- 掌握局部变量和全局变量的概念及作用域
- 理解变量的存储类别（auto、static、register、extern）
- 掌握静态局部变量的特性和使用方法
- 理解变量的作用域和生存期的区别


## 📖 知识梳理

### 7.4.1 局部变量和全局变量 

从变量的**作用域**（即从空间的角度）来观察，变量可以分为**局部变量**和**全局变量**。

#### 一、局部变量

在一个函数内部或复合语句中定义的变量，只在本函数或复合语句范围内有效（从定义点开始，到函数或复合语句结束），它们被称为**局部变量**。

> 💡 **核心要点**：局部变量只在定义它的函数或复合语句内有效，离开该范围后不能被访问。

**局部变量的有效范围：**

```c
float f1(int a)              // 形参a也是局部变量
{
    int b, c;                // b、c只在f1函数内有效
    ┌─────────────────────────────┐
    │  a、b、c的有效范围           │
    └─────────────────────────────┘
}

int main()
{
    int m, n;                // m、n只在main函数内有效
    ┌─────────────────────────────────────┐
    │  m、n的有效范围                     │
    │  ┌─────────────────────────────┐    │
    │  │ int c;                     │    │
    │  │ c = m + n;                 │    │
    │  │ c在此复合语句内有效         │    │
    │  └─────────────────────────────┘    │
    └─────────────────────────────────────┘
}
```

**局部变量的特点：**

| 特点 | 说明 |
| :--- | :--- |
| ① 作用域限制 | 只在定义它的函数或复合语句内有效 |
| ② 不同函数同名变量 | 不同函数中可以使用相同的变量名，它们代表不同的对象，互不干扰 |
| ③ 形式参数 | 形参也是局部变量，只在函数内有效 |
| ④ 复合语句变量 | 在复合语句中定义的变量，只在该复合语句内有效 |

> 💡 **即学即练**：
> - （判断）一个函数内的复合语句中定义的变量，只在函数内部有效。 **✖**
> - （判断）在不同函数中可以使用相同名称的变量。 **✔**

#### 二、全局变量

在**函数之外定义的变量**称为**全局变量**（又称外部变量）。

**全局变量的有效范围：**

```c
int p = 1, q = 5;            // 定义外部变量p、q
                              // p、q的有效范围：从定义处到文件结束

float f1(int a)              // f1函数只能使用p、q，不能使用c1、c2
{
    int b, c;
    ...
}

char c1, c2;                 // 定义外部变量c1、c2
                              // c1、c2的有效范围：从定义处到文件结束

char f2(int x, int y)        // f2函数可以使用p、q、c1、c2
{
    int i, j;
    ...
}

int main()                   // main函数可以使用p、q、c1、c2
{
    int m, n;
    ...
}
```

**全局变量的特点：**

| 特点 | 说明 |
| :--- | :--- |
| ① 作用域 | 从定义处开始，到本程序文件的末尾 |
| ② 共享性 | 在作用域内的所有函数都可以使用全局变量 |
| ③ 数据传递 | 如果在某个函数中改变了全局变量的值，会影响其他函数中该变量的值 |
| ④ 多值传递 | 通过全局变量可以在函数间传递多个值 |


### 7.4.2 变量的存储方式和生存期

从变量值存在的**时间**（即生存期）来观察，变量的存储有两种不同的方式：**静态存储方式**和**动态存储方式**。

#### 一、存储方式分类

| 存储方式 | 说明 | 适用变量 |
| :--- | :--- | :--- |
| **静态存储方式** | 在程序运行期间由系统分配**固定的存储空间**，程序执行完毕才释放 | 全局变量、静态局部变量 |
| **动态存储方式** | 在程序运行期间**根据需要随时分配**存储空间，函数调用结束时释放 | 自动变量、形参 |

**存储方式示意图：**

```
┌─────────────────────────────────────────────────────┐
│                    内存                              │
│  ┌─────────────────────────────────────────────────┐ │
│  │             静态存储区                          │ │
│  │  ┌───────────┐ ┌───────────┐ ┌───────────┐   │ │
│  │  │ 全局变量  │ │静态局部变量│ │ 静态外部  │   │ │
│  │  └───────────┘ └───────────┘ └───────────┘   │ │
│  │  （程序开始分配，程序结束释放）                  │ │
│  └─────────────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────────────┐ │
│  │             动态存储区                          │ │
│  │  ┌───────────┐ ┌───────────┐                   │ │
│  │  │ 自动变量  │ │ 形参      │                   │ │
│  │  └───────────┘ └───────────┘                   │ │
│  │  （函数调用时分配，函数结束时释放）              │ │
│  └─────────────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────────────┐ │
│  │             寄存器                              │ │
│  │  ┌───────────┐                                 │ │
│  │  │寄存器变量  │                                 │ │
│  │  └───────────┘                                 │ │
│  └─────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

#### 二、变量的存储类别

C语言提供了4种存储类别：

| 存储类别 | 关键字 | 存储位置 | 生存期 |
| :--- | :--- | :--- | :--- |
| **自动变量** | `auto` | 动态存储区 | 函数调用期间 |
| **静态变量** | `static` | 静态存储区 | 程序运行期间 |
| **寄存器变量** | `register` | CPU寄存器 | 函数调用期间 |
| **外部变量** | `extern` | 静态存储区 | 程序运行期间 |

**（1）自动变量（auto变量）**

函数中的形参和在函数中定义的局部变量（包括在复合语句中定义的局部变量），都属于**自动变量**。

```c
int f(int a)                 // a是形参，属于自动变量
{
    auto int b, c = 3;       // b、c是自动变量，c赋初值3
    ...
}
// 执行完f函数后，自动释放a、b、c所占的存储单元
```

| 特点 | 说明 |
| :--- | :--- |
| 分配时机 | 函数调用时分配存储空间 |
| 释放时机 | 函数调用结束时自动释放 |
| 初值 | 如果不赋初值，其值是**不确定的** |
| 关键字 | `auto` 可以省略（默认） |

**（2）静态局部变量（static局部变量）**

如果希望函数中的局部变量的值在函数调用结束后**不消失**而继续保留原值，则应指定该局部变量为**静态局部变量**，用关键字 `static` 进行声明。

**示例：考察静态局部变量的值**

```c
#include <stdio.h>

int main()
{
    int f(int);              // 函数声明
    int a = 2, i;
    for (i = 0; i < 3; i++)
    {
        printf("%d\n", f(a));
    }
    return 0;
}

int f(int a)
{
    auto int b = 0;          // 自动局部变量，每次调用初值为0
    static int c = 3;        // 静态局部变量，只在第一次调用时初始化
    b = b + 1;
    c = c + 1;
    return (a + b + c);
}
```

**运行结果：**

```
7
8
9
```

**程序分析：**

| 调用次数 | b的初值 | c的初值 | 调用结束时b的值 | 调用结束时c的值 | 返回值(a+b+c) |
| :---: | :---: | :---: | :---: | :---: | :---: |
| 第1次 | 0 | 3 | 1 | 4 | 2+1+4=7 |
| 第2次 | 0 | 4 | 1 | 5 | 2+1+5=8 |
| 第3次 | 0 | 5 | 1 | 6 | 2+1+6=9 |

> 💡 **核心区别**：
> - `b`（自动变量）：每次调用时**重新初始化**为0
> - `c`（静态局部变量）：**只初始化一次**，保留上次调用结束时的值

**静态局部变量与自动变量的区别：**

| 对比项 | 静态局部变量（static） | 自动变量（auto） |
| :--- | :--- | :--- |
| 存储位置 | 静态存储区 | 动态存储区 |
| 初始化 | 编译时赋初值，**只初始化一次** | 函数调用时赋初值，**每次调用都初始化** |
| 未赋初值 | 自动赋初值 **0** | 值是**不确定的** |
| 函数结束后 | 值**保留**，但其他函数不能访问 | 值**消失**，存储单元被释放 |
| 生存期 | 程序运行期间 | 函数调用期间 |

**（3）寄存器变量（register变量）**

一般情况下，变量的值是存放在内存中的。如果有一些变量使用频繁，为提高执行效率，允许将局部变量存储在CPU的**寄存器**中，这种变量叫作**寄存器变量**，用关键字 `register` 说明。

```c
register int f;              // 定义寄存器变量
```

| 特点 | 说明 |
| :--- | :--- |
| 优点 | 存取速度快，提高执行效率 |
| 适用场景 | 使用频繁的变量（如循环变量） |
| 限制 | 不能取地址（`&`），因为寄存器没有地址 |

**（4）外部变量（extern变量）**

一般来说，外部变量是在函数的外部定义的全局变量，它的作用域是从变量的定义处开始，到本程序文件的末尾。

**扩展外部变量作用域的两种情况：**

| 情况 | 方法 | 说明 |
| :--- | :--- | :--- |
| ① 在**一个文件内**扩展 | 在引用前用 `extern` 声明 | 将外部变量的作用域扩展到声明处之前 |
| ② 扩展到**其他文件** | 在另一文件中用 `extern` 声明 | 声明该变量是在其他文件中定义的外部变量 |


### 7.4.3 作用域和生存期小结

#### 一、各种类型变量的作用域和生存期

| 变量存储类别 | 函数内可见性 | 函数内存在性 | 函数外可见性 | 函数外存在性 |
| :--- | :---: | :---: | :---: | :---: |
| 自动变量 / 寄存器变量 | ✔ | ✔ | ✖ | ✖ |
| 静态局部变量 | ✔ | ✔ | ✖ | ✔ |
| 静态外部变量 | ✔ | ✔ | ✔（本文件） | ✔ |
| 外部变量（非静态） | ✔ | ✔ | ✔ | ✔ |

> 💡 **理解**：
> - **可见性**：变量是否可以被访问
> - **存在性**：变量是否在内存中存在
> - 静态局部变量：**存在但不可见**（函数外存在，但不可被访问）

#### 二、从两个角度理解变量

```
         ┌─────────────────────────────────────────────┐
         │              变量属性                       │
         ├─────────────────────┬───────────────────────┤
         │   作用域（空间）     │   生存期（时间）      │
         ├─────────────────────┼───────────────────────┤
         │  局部变量           │  动态存储             │
         │  （函数/复合语句内） │  （函数调用期间）     │
         ├─────────────────────┼───────────────────────┤
         │  全局变量           │  静态存储             │
         │  （文件范围内）     │  （程序运行期间）     │
         └─────────────────────┴───────────────────────┘
```


## 💡 核心考点

1. **局部变量**：函数内部定义，作用域仅限于该函数
2. **全局变量**：函数外部定义，作用域从定义处到文件结束
3. **静态存储方式**：程序运行期间固定分配存储空间
4. **动态存储方式**：函数调用时分配，调用结束释放
5. **auto变量**：默认存储类别，函数调用时分配，结束后释放
6. **static局部变量**：静态存储，只初始化一次，函数结束后值保留
7. **register变量**：存储在CPU寄存器中，存取速度快
8. **extern变量**：扩展外部变量的作用域
