# 7.1 函数的概念与定义函数

## 🎯 学习目标

- 理解函数在模块化程序设计中的作用

## 📖 知识梳理

### 7.1.1 函数及其分类

#### 一、模块化程序设计思想

在实际应用中，为了降低大规模软件的开发难度，程序员必须把较大的任务分解成若干个较简单的任务，并提炼出公用任务，这种方法称为**分而治之**。模块化程序设计就体现了这种"分而治之"的思想。

**功能分解的过程：**

```
┌─────────────────────────────────────────┐
│          自顶向下、逐步求精              │
│                                          │
│     ┌─────────────────────────┐         │
│     │      大功能（总任务）    │         │
│     └───────────┬─────────────┘         │
│                   ↓                       │
│     ┌───────────┴───────────┐            │
│     ↓           ↓           ↓            │
│ ┌───────┐ ┌───────┐ ┌───────┐          │
│ │ 子功能 │ │ 子功能 │ │ 子功能 │          │
│ └───┬───┘ └───┬───┘ └───┬───┘          │
│     ↓         ↓         ↓                │
│ ┌───┴───┐ ┌───┴───┐ ┌───┴───┐          │
│ │子子功能│ │子子功能│ │子子功能│          │
│ └───────┘ └───────┘ └───────┘          │
└─────────────────────────────────────────┘
```

**模块化程序设计的优点：**

| 优点 | 说明 |
| :--- | :--- |
| **易于理解** | 每个模块功能单一，逻辑清晰 |
| **易于调试** | 可以独立测试每个模块 |
| **易于维护** | 修改某个模块不影响其他模块 |
| **代码复用** | 同一函数可在多处调用 |
| **团队协作** | 不同开发者可同时开发不同模块 |

#### 二、函数的概念

**函数**就是功能，函数名就是给该功能起的名字。每一个函数用来实现一个**特定的功能**。

- C语言中，**函数是模块化程序设计的最小单位**
- 既可以把每个函数都看作一个模块
- 也可以将若干相关的函数合并成一个模块

**C程序的结构：**

```
┌─────────────────────────────────────────────────────┐
│                    C程序                            │
│  ┌─────────────────────────────────────────────────┐│
│  │             源程序文件1                        ││
│  │  ┌─────────┐ ┌─────────┐ ┌─────────┐         ││
│  │  │ 函数1   │ │ 函数2   │ │ 函数3   │         ││
│  │  └─────────┘ └─────────┘ └─────────┘         ││
│  └─────────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────────┐│
│  │             源程序文件2                        ││
│  │  ┌─────────┐ ┌─────────┐                     ││
│  │  │ 函数4   │ │ 函数5   │                     ││
│  │  └─────────┘ └─────────┘                     ││
│  └─────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘
```

#### 三、函数的分类

从不同角度，函数可以有不同的分类方式：

| 分类标准 | 分类 | 说明 |
| :--- | :--- | :--- |
| **从用户使用角度** | **库函数** | C编译系统提供的、预先定义好的函数，如 `printf`、`scanf` |
| | **用户自定义函数** | 程序员根据需求自己定义的函数 |
| **从函数形式** | **无参函数** | 调用时不需要传递参数 |
| | **有参函数** | 调用时需要传递参数 |

**库函数与自定义函数的区别：**

| 对比项 | 库函数 | 用户自定义函数 |
| :--- | :--- | :--- |
| **定义者** | C编译系统 | 程序员 |
| **是否需定义** | 无需定义，直接使用 | 必须先定义后使用 |
| **头文件** | 需要包含对应的头文件 | 无需包含额外头文件 |
| **典型示例** | `printf`、`scanf`、`sqrt` | 用户编写的 `max`、`min` 等 |


### 7.1.2 定义函数 

#### 一、定义函数的概念

C语言要求在程序中用到的所有函数，必须**"先定义，后使用"**。

**为什么要先定义后使用？**

以定义求最小值的 `min` 函数为例：必须事先按规范对 `min` 进行定义，指定它的名字、函数返回值类型、参数个数与类型等信息。这样，在程序执行到 `min` 函数时，编译系统就会按照定义时所指定的功能执行。如果事先不定义，编译系统就不知道 `min` 是什么、要实现什么功能。

**定义函数应包括以下内容：**

| 内容 | 说明 |
| :--- | :--- |
| ① **函数名** | 指定函数的名称，以便以后按名调用 |
| ② **函数类型** | 指定函数返回值的类型 |
| ③ **参数列表** | 指定函数的参数的名字和类型，以便在调用函数时向它们传递数据 |
| ④ **函数体** | 指定函数应当完成什么操作，即函数的功能（最重要的部分） |

#### 二、定义函数的方法

**1. 无参函数**

```c
类型名 函数名()
{
    声明部分;
    执行语句部分;
}
```

**示例：**
```c
void print_star()
{
    printf("********************\n");
}
```

**2. 有参函数**

```c
类型名 函数名(形参列表)
{
    声明部分;
    执行语句部分;
    return 表达式;
}
```

**示例：**
```c
int min(int x, int y)    // 定义有参函数min，求x和y中的最小值
{
    int z;               // 声明部分
    if (x < y)
        z = x;
    else
        z = y;
    return z;            // 返回z的值
}
```

**3. 函数定义的各部分详解**

```c
int min(int x, int y)    // 第1行：函数首部
{                          // 第2行：函数体开始
    int z;               // 第3行：声明部分（定义局部变量）
    if (x < y)           // 第4行：执行语句部分
        z = x;
    else
        z = y;
    return z;            // 第8行：返回语句
}                          // 第9行：函数体结束
```

| 行号 | 内容 | 说明 |
| :---: | :--- | :--- |
| 第1行 | `int min(int x, int y)` | 函数首部：返回值类型 + 函数名 + 形参列表 |
| 第3行 | `int z;` | 声明部分：定义函数内部使用的变量 |
| 第4-8行 | `if...else...return` | 执行语句部分：实现函数的具体功能 |
| 第8行 | `return z;` | 返回语句：将结果返回给调用者 |

#### 三、函数类型说明

| 函数类型 | 关键字 | 说明 |
| :--- | :--- | :--- |
| **有返回值** | `int`、`float`、`char` 等 | 函数执行完毕后返回一个值 |
| **无返回值** | `void` | 函数不需要返回任何值 |

> ⚠️ **注意**：若指定函数类型为 `void`，表示函数**无类型**，不需要带回任何函数值。

#### 四、形参（形式参数）

形参出现在**函数定义**中，用于接收调用函数时传递过来的数据。

| 要点 | 说明 |
| :--- | :--- |
| **位置** | 在函数首部的括号内定义 |
| **个数** | 可以有一个或多个，也可以没有 |
| **作用** | 接收实参传递的数据 |
| **类型** | 必须指定每个形参的数据类型 |

**示例：**
```c
int max(int a, int b, int c)    // 3个形参：a、b、c
{
    // 函数体
}

void print()                    // 无参函数
{
    // 函数体
}
```


## 💡 核心考点

1. **模块化程序设计**：自顶向下、逐步求精、分而治之
2. **函数的概念**：实现特定功能的程序模块
3. **函数分类**：库函数 vs 用户自定义函数；有参函数 vs 无参函数
4. **使用原则**：先定义，后使用
5. **函数定义四要素**：函数名、函数类型、参数列表、函数体
6. **无参函数**：`类型名 函数名() { 函数体 }`
7. **有参函数**：`类型名 函数名(形参列表) { 函数体 }`
8. **void类型**：表示函数无返回值
9. **形参**：定义在函数首部，用于接收实参传递的数据
