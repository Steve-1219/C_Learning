## 📖 知识梳理

### 1.1.1 算法和程序

#### 一、基本概念

计算机的"自动"工作是由程序控制的。**程序**就是一组计算机能识别和执行的指令，每一条指令使计算机执行特定的操作。要让计算机按照人们的意愿工作，必须将事先编制好的程序输入计算机中。

一个程序包括以下两方面的内容：

| 内容             | 说明                                                         |
| :--------------- | :----------------------------------------------------------- |
| **对数据的描述** | 在程序中要指定用到哪些数据以及这些数据的类型和数据的组织形式，即**数据结构** |
| **对操作的描述** | 即要求计算机进行操作的步骤，也就是**算法**，用于解决"做什么"和"怎么做"的问题 |

数据是操作的对象，操作的目的是对数据进行加工处理，以得到期望的结果。著名的计算机科学家沃思曾提出一个公式：

> **算法 + 数据结构 = 程序**

这是对面向过程程序的概括。一个程序除了以上两个主要要素外，还应当采用适当的**程序设计方法**，并且用一种**计算机语言**进行表示。因此，在设计程序时要综合考虑以下四个方面：

| 要素         | 作用     |
| :----------- | :------- |
| 算法         | 灵魂     |
| 数据结构     | 加工对象 |
| 语言工具     | 工具     |
| 程序设计方法 | 方法     |

#### 二、算法的特性

| 特性       | 说明                                                     |
| :--------- | :------------------------------------------------------- |
| **输入**   | 可以有零个或多个输入，执行算法时需要从外界获取必要的信息 |
| **输出**   | 至少有一个或多个输出，算法的目的就是求解，"解"就是输出   |
| **有穷性** | 算法在执行有限步后必须结束，不能无限循环                 |
| **确定性** | 算法的每一个步骤都具有确定的含义，不能出现二义性         |
| **可行性** | 算法的每一步都必须是可以执行的                           |

#### 三、算法的表示方法

算法的表示方法有很多，常用的有：自然语言、流程图、伪代码、计算机语言等。

##### （1）自然语言

- **优点**：通俗易懂
- **缺点**：文字冗长，容易出现歧义，需要根据上下文才能判断其正确含义
- **适用**：除了很简单的问题以外，一般不使用

##### （2）流程图

用一些图框来表示各种操作。ANSI（美国国家标准协会）规定了一些常用的流程图符号：

| 符号名称   | 图形 | 功能                     |
| :--------- | :--- | :----------------------- |
| 起止框     | ⟡    | 表示算法的开始和结束     |
| 输入输出框 | ◇    | 表示算法的输入、输出操作 |
| 处理框     | ▭    | 表示算法中的各种处理操作 |
| 判断框     | ◇    | 表示算法中的条件判断操作 |
| 流程线     | →    | 表示算法的执行方向       |
| 连接点     | ○    | 表示流程图的延续         |

- **优点**：直观形象，易于理解
- **缺点**：画图比较麻烦，修改比较困难

##### （3）伪代码

用介于自然语言和计算机语言之间的文字符号来描述算法。伪代码无固定的、严格的语法规则，可以用英文，也可以中英文混用，只要把意思表达清楚即可。

**【例1】用伪代码表示"输出x的绝对值"的算法**

```
方法一（英文）：
if x is positive then
    print x
else
    print -x

方法二（中英混用）：
若 x 为正
    输出 x
否则
    输出 -x
```

##### （4）计算机语言

计算机无法识别自然语言、流程图、伪代码。这些方法仅为了帮助人们描述、理解算法，要用计算机解题，就必须用计算机语言描述算法。只有用计算机语言编写的程序才能被计算机执行（还要经过编译转换成目标程序）。

**【例2】计算 1×2×3×4×5**

用流程图表示：

```
       开始
         ↓
    t = 1, i = 2
         ↓
   ┌─ t = t × i ─┐
   │    i = i + 1 │
   └──────┬───────┘
          ↓
      i > 5 ？
       /    \
      是     否
       ↓
    输出 t
       ↓
     结束
```

用C语言表示：

```c
#include <stdio.h>
int main() {
    int t = 1, i = 2;
    while (i <= 5) {
        t = t * i;
        i = i + 1;
    }
    printf("%d\n", t);
    return 0;
}
```


### 1.1.2 程序的三种基本结构 

一个程序包含一系列的执行语句，每一个语句使计算机完成特定的操作。程序设计时应当遵循结构化程序设计的原则，尽量避免不必要的跳转，使各语句按照从上到下的顺序执行。

程序的三种基本结构包括：

| 结构         | 说明                               | C语言关键字                    |
| :----------- | :--------------------------------- | :----------------------------- |
| **顺序结构** | 按照书写顺序依次执行各语句         | 无特定关键字                   |
| **选择结构** | 根据条件判断选择执行不同的语句分支 | `if`、`else`、`switch`、`case` |
| **循环结构** | 在满足条件时重复执行某段代码       | `while`、`do-while`、`for`     |

这三种基本结构是程序设计的基础，任何复杂的程序都可以由这三种结构组合而成。

#### 三种结构的特点

- **单入口单出口**：每种结构都只有一个入口和一个出口
- **结构清晰**：便于阅读、理解和维护
- **组合灵活**：三种结构可以相互嵌套、组合


### 1.1.3 结构化程序设计

#### 一、基本思想

结构化程序设计是一种经典的编程方法论，其核心思想是：**任何程序都可以用顺序、选择、循环三种基本结构来表示**。

1966年，计算机科学家Böhm和Jacopini证明了：任何复杂的程序都可以由这三种基本结构通过组合和嵌套来实现，而不需要使用 `goto` 语句。

#### 二、三种基本结构的详细介绍

##### （1）顺序结构（Sequence Structure）

**定义**：按照代码书写的先后顺序，从上到下依次执行各条语句。

**特点**：
- 程序执行的唯一流向，无分支、无跳转
- 是最基本、最简单的结构
- 任何程序都包含顺序结构

**流程图表示**：

```
┌─────────────┐
│   语句A     │
├─────────────┤
│   语句B     │
├─────────────┤
│   语句C     │
└─────────────┘
```

**C语言示例**：

```c
int a = 5;          // 语句1：赋值
a = a + 3;          // 语句2：运算
printf("%d", a);    // 语句3：输出
// 按顺序执行：语句1 → 语句2 → 语句3
```

##### （2）选择结构（Selection Structure）

**定义**：根据给定的条件进行判断，选择执行不同的语句分支。

**分类**：

| 类型           | 说明                               | C语言关键字       |
| :------------- | :--------------------------------- | :---------------- |
| **单分支选择** | 条件成立时执行，不成立时跳过       | `if`              |
| **双分支选择** | 条件成立时执行A，不成立时执行B     | `if-else`         |
| **多分支选择** | 根据表达式的值选择执行多个分支之一 | `switch-case`     |
| **嵌套选择**   | 在选择结构内部再包含选择结构       | `if-else if-else` |

**流程图表示**：

```
        ┌──────────────┐
        │  条件P成立？  │
        └──────┬───────┘
          成立 /    \ 不成立
              /      \
        ┌─────┐    ┌─────┐
        │  A  │    │  B  │
        └─────┘    └─────┘
              \      /
               \    /
            ┌─────┐
            │ 后续  │
            └─────┘
```

**C语言示例**：

```c
// 单分支选择
if (score >= 60) {
    printf("及格\n");
}

// 双分支选择
if (score >= 60) {
    printf("及格\n");
} else {
    printf("不及格\n");
}

// 多分支选择
switch (grade) {
    case 'A': printf("优秀\n"); break;
    case 'B': printf("良好\n"); break;
    case 'C': printf("及格\n"); break;
    default: printf("不及格\n");
}
```

##### （3）循环结构（Loop Structure）

**定义**：在满足一定条件时，重复执行某段代码块。

**分类**：

| 类型           | 说明                         | C语言关键字 | 特点             |
| :------------- | :--------------------------- | :---------- | :--------------- |
| **当型循环**   | 先判断条件，成立时执行循环体 | `while`     | 可能一次都不执行 |
| **直到型循环** | 先执行循环体，再判断条件     | `do-while`  | 至少执行一次     |
| **计数型循环** | 指定循环次数的循环           | `for`       | 适合已知循环次数 |

**流程图表示**：

```
当型循环（while）：      直到型循环（do-while）：
┌──────────────┐       ┌──────────────┐
│  当条件P成立  │       │  循环体A     │
│      │       │       │      │       │
│  循环体A     │       │  当条件P成立  │
└──────────────┘       └──────────────┘
```

**C语言示例**：

```c
// 当型循环（while）
int i = 1;
while (i <= 5) {
    printf("%d ", i);
    i++;
}

// 直到型循环（do-while）
int j = 1;
do {
    printf("%d ", j);
    j++;
} while (j <= 5);

// 计数型循环（for）
for (int k = 1; k <= 5; k++) {
    printf("%d ", k);
}
```

#### 三、三种结构的对比

| 对比项           | 顺序结构 | 选择结构           | 循环结构             |
| :--------------- | :------- | :----------------- | :------------------- |
| **执行方式**     | 依次执行 | 条件判断后选择执行 | 条件控制重复执行     |
| **执行次数**     | 一次     | 一次               | 零次或多次           |
| **是否改变流向** | 否       | 是（分支）         | 是（重复）           |
| **C语言关键字**  | 无       | `if`、`switch`     | `while`、`for`、`do` |
| **复杂度**       | 最低     | 中等               | 较高                 |

#### 四、三种结构的组合方式

三种基本结构通过以下方式组合成复杂程序：

| 组合方式     | 说明                           | 示例                           |
| :----------- | :----------------------------- | :----------------------------- |
| **顺序组合** | 一个结构执行完后执行另一个结构 | 顺序结构 → 选择结构 → 循环结构 |
| **嵌套组合** | 一个结构内部包含另一个结构     | 循环内部嵌套选择结构           |
| **并列组合** | 多个相同结构并列存在           | 多个 `if` 语句并列             |

**嵌套示例**：循环内嵌套选择结构

```c
for (int i = 1; i <= 10; i++) {      // 循环结构（外层）
    if (i % 2 == 0) {               // 选择结构（内层）
        printf("%d 是偶数\n", i);
    } else {
        printf("%d 是奇数\n", i);
    }
}
```

#### 五、结构化程序设计的优点

| 优点         | 说明                         |
| :----------- | :--------------------------- |
| **可读性好** | 结构清晰，便于他人阅读和理解 |
| **易于调试** | 模块独立，容易定位错误       |
| **便于维护** | 修改某一部分不影响其他部分   |
| **可靠性高** | 结构规范，减少程序错误       |
| **团队开发** | 功能模块可并行开发           |


### 1.1.4 程序设计语言的发展和常见的高级语言

#### 一、程序设计语言的发展阶段

| 阶段       | 语言类型     | 特点                           | 代表语言           |
| :--------- | :----------- | :----------------------------- | :----------------- |
| **第一代** | 机器语言     | 二进制代码，计算机直接执行     | 无特定名称         |
| **第二代** | 汇编语言     | 助记符代替二进制，需汇编器翻译 | ASM                |
| **第三代** | 高级语言     | 接近自然语言，需编译或解释执行 | C、Pascal、Fortran |
| **第四代** | 非过程化语言 | 面向问题，只需描述"做什么"     | SQL                |
| **第五代** | 人工智能语言 | 基于逻辑和约束                 | Prolog、Lisp       |

#### 二、常见的高级语言

| 语言           | 特点                 | 主要应用领域                   |
| :------------- | :------------------- | :----------------------------- |
| **C语言**      | 高效、灵活、面向过程 | 操作系统、嵌入式、系统软件     |
| **C++**        | 面向对象、兼容C      | 系统软件、游戏开发、高性能计算 |
| **Java**       | 跨平台、面向对象     | Web开发、企业级应用、Android   |
| **Python**     | 简洁易学、库丰富     | 数据分析、人工智能、Web开发    |
| **C#**         | 面向对象、.NET平台   | Windows应用、游戏开发          |
| **JavaScript** | 动态类型、事件驱动   | 前端开发、Node.js后端          |
| **Go**         | 并发支持、编译快     | 云原生、微服务、网络编程       |
| **Rust**       | 内存安全、高性能     | 系统编程、WebAssembly          |

#### 三、C语言的地位

C语言是程序设计语言中的"基石"：

- 是学习其他高级语言（C++、Java、C#等）的基础
- 广泛应用于操作系统（Unix/Linux）、嵌入式系统、驱动程序开发
- 高效、灵活，能直接操作硬件和内存
- 具备高级语言的可读性和汇编语言的效率


## 💡 核心考点

1. **沃思公式**：算法 + 数据结构 = 程序（常考填空题）
2. **算法的5个特性**：输入、输出、有穷性、确定性、可行性
3. **算法的表示方法**：自然语言、流程图、N-S图、伪代码、计算机语言
4. **程序的三种基本结构**：顺序结构、选择结构、循环结构
5. **结构化程序设计的核心**：自顶向下、逐步求精、模块化
6. **程序设计语言的发展阶段**：机器语言 → 汇编语言 → 高级语言

| **自顶向下**           | 从总体问题出发，逐步分解为子问题       |
| **逐步求精**           | 将复杂问题逐步细化为具体步骤           |
| **模块化**             | 将程序划分为功能相对独立的模块         |
| **限制使用 goto 语句** | 尽量使用三种基本结构，避免程序跳转混乱 |
