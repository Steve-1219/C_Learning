# 1.3 运行C程序的步骤与方法

## 🎯 学习目标

- 掌握C程序从编写到运行的四个基本步骤
- 理解编辑、编译、链接、运行各阶段的作用
- 学会在CLion和VS Code中运行C语言程序
- 了解常见的编译错误类型及处理方法


## 📖 知识梳理

### 1.3.1 运行C程序的步骤 

一个C程序从编写到得到运行结果，需要经历以下四个基本步骤：

```
编辑源程序 → 编译 → 链接 → 运行
    ↓          ↓      ↓      ↓
  .c文件 →  .obj文件 → .exe → 输出结果
```

#### 一、编辑（Edit）

**编辑**就是使用文本编辑器或集成开发环境（IDE）编写C语言源代码。

- 源文件以 `.c` 为扩展名（如 `hello.c`）
- 这是整个过程的起点，程序员在此编写程序指令

#### 二、编译（Compile）

**编译**是将源程序翻译成计算机能够识别的目标代码的过程。

- 编译程序（编译器）对源程序进行**词法检查**和**语法检查**
- 编译以源程序文件为单位分别进行
- 编译成功后生成目标文件，扩展名为 `.obj`（Windows）或 `.o`（Unix/Linux）
- 如果编译失败，编译器会显示错误（error）和警告（warning）信息

> **编译的作用**：将高级语言（C语言）转换为机器语言，使计算机能够理解程序指令。

#### 三、链接（Link）

**链接**是将一个程序的所有目标文件和系统的库文件连接在一起，形成一个可执行文件的过程。

- 使用系统提供的**链接程序（Linker）**完成
- 将多个目标文件（`.obj`）和库函数文件合并
- 最终生成可执行文件，Windows下为 `.exe` 文件
- 如果链接失败，需要检查库文件引用是否正确

> **链接的作用**：解决程序中引用的外部函数（如 `printf`）和变量的定义位置问题，将它们与程序代码关联起来。

#### 四、运行（Run）

**运行**是指执行生成的可执行文件，得到最终的程序运行结果。

- 在Windows下可以直接运行 `.exe` 文件
- 在Unix/Linux下，键入可执行文件名即可
- 如果运行结果不正确，需要返回编辑阶段检查算法或代码逻辑

#### 各步骤常见错误

| 步骤 | 可能出现的错误                 | 处理方法                   |
| :--- | :----------------------------- | :------------------------- |
| 编辑 | 拼写错误、漏写分号、括号不匹配 | 检查并修改源代码           |
| 编译 | 语法错误、类型不匹配           | 根据编译器提示修改         |
| 链接 | 找不到库文件、重复定义         | 检查 `#include` 和函数声明 |
| 运行 | 逻辑错误、运行时错误（如除零） | 检查算法逻辑，添加边界检查 |

> 💡 **提醒**：在实际开发中，经常需要循环往复地执行这些步骤。如果程序运行结果不正确，需要返回编辑阶段重新修改代码，然后再次编译、链接、运行，直到得到正确结果。


### 1.3.2 上机运行C语言的方法

目前运行C语言程序主要有两种方式：使用**集成开发环境（IDE）** 和使用**代码编辑器 + 编译器**。本节以 **CLion** 和 **VS Code** 为例进行介绍。

#### 一、使用CLion运行C程序

CLion是JetBrains公司推出的专业C/C++集成开发环境，功能强大，适合系统学习。对于初学者，CLion支持不创建完整项目直接运行单个文件，非常便捷。

**1. 准备工具链**

首次使用CLion需要配置编译器：
- **Windows**：MinGW-w64 或 Microsoft Visual C++ 工具链
- **macOS**：Xcode Command Line Tools（自带Clang/GCC）
- **Linux**：GCC 或 Clang

CLion会自动检测系统中的编译器。

**2. 打开单个文件（推荐初学者）**

对于刚学习C语言、只有一两个文件的情况，不需要创建完整项目：

1. 点击欢迎屏幕上的 **Open**（打开）
2. 选择包含 `.c` 文件的文件夹
3. CLion会以无项目模型的方式打开文件

**3. 编写代码**

在编辑器中编写C语言程序，例如：

```c
#include <stdio.h>

int main() {
    printf("Hello, CLion!\n");
    return 0;
}
```

**4. 运行程序**

- 在 `main` 函数左侧的边栏中，会出现一个绿色三角形图标
- 点击该图标，选择 **Run**，文件将被编译并运行
- 运行结果会显示在 **Run** 工具窗口中

**5. 调试程序**

如果需要调试：
- 在代码行号左侧点击设置断点
- 点击侧栏图标选择 **Debug**
- CLion会启动调试会话，可以单步执行查看变量值

**快速上手总结：**

```
打开文件 → 编写代码 → 点击侧栏Run → 查看输出
```


#### 二、使用VS Code运行C程序

VS Code是轻量级的代码编辑器，通过安装扩展可以实现C语言的编译和运行。

**1. 安装编译器**

VS Code本身不具备编译功能，需要先安装C编译器：

- **Windows**：安装 MinGW-w64（GCC编译器移植版）
- **macOS**：执行 `xcode-select --install`
- **Linux**：执行 `sudo apt install build-essential gdb`（Ubuntu/Debian）

安装完成后，在命令行执行 `gcc --version` 验证是否成功。

**2. 安装VS Code扩展**

打开VS Code，在扩展商店中搜索并安装以下插件：

| 插件                       | 用途                         |
| :------------------------- | :--------------------------- |
| **C/C++**（Microsoft官方） | 语法高亮、代码补全、调试支持 |
| **Code Runner**            | 一键编译运行代码             |

**3. 编写代码**

创建一个新文件，命名为 `hello.c`，编写C程序：

```c
#include <stdio.h>

int main() {
    printf("Hello, VS Code!\n");
    return 0;
}
```

**4. 运行程序的三种方法**

| 方法                | 操作                                                         |
| :------------------ | :----------------------------------------------------------- |
| **Code Runner插件** | 右键点击代码文件选择 **Run Code**，或按 `Ctrl+Alt+N`         |
| **手动编译运行**    | 按 `Ctrl+` 打开终端，执行 `gcc hello.c -o hello`，然后运行 `./hello`（或 `hello.exe`） |
| **点击运行按钮**    | 点击右上角的三角形运行按钮（需安装Code Runner）              |

**5. 实时错误检查**

安装 **C/C++** 扩展后，保存文件（`Ctrl+S`）时会自动进行语法检查，错误会以红色波浪线显示。

**快速上手总结：**

```
安装MinGW → 安装C/C++和Code Runner扩展 → 编写代码 → 点击Run运行
```


#### 三、两种工具的对比

| 对比项         | CLion               | VS Code                     |
| :------------- | :------------------ | :-------------------------- |
| **类型**       | 集成开发环境（IDE） | 代码编辑器                  |
| **编译器配置** | 自动检测，配置简单  | 需手动安装MinGW             |
| **运行方式**   | 点击侧栏绿色按钮    | 点击右上角Run按钮或终端命令 |
| **调试支持**   | 内置强大调试器      | 需要配置 launch.json        |
| **适合场景**   | 专业开发、大型项目  | 轻量开发、快速练习          |
| **价格**       | 收费（学生免费）    | 免费                        |


## 💡 核心考点

1. **C程序运行四步骤**：编辑 → 编译 → 链接 → 运行
2. **编译的作用**：将C源程序翻译成目标代码（`.obj`/`.o`文件）
3. **链接的作用**：将目标文件和库文件合并为可执行文件（`.exe`）
4. **源文件扩展名**：`.c`；目标文件扩展名：`.obj`；可执行文件扩展名：`.exe`
5. **CLion运行方式**：打开文件 → 点击侧栏绿色三角形 → Run
6. **VS Code运行方式**：安装MinGW → 安装C/C++和Code Runner扩展 → 点击Run
